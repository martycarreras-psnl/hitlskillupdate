Power Apps Code App · Human-in-the-Loop

Reviewers don't just fix data —
they upgrade the AI.

A closed feedback loop for document intake. An AI agent extracts structured data from uploaded documents at scale; a configurable sample plus rule-based flags route records to a human reviewer; and every rejection becomes an edit to the live agent skill — so the next run is measurably better than the last.

The problem we solve

Capture the lesson, not just the correction

Document-extraction agents are never perfect. The usual answer is to have a human fix the output — which fixes one record but loses the lesson. This solution captures the lesson and feeds it back into the agent's skills, stored in Dataverse, so quality compounds over time.

End to end

How it works, step by step

Ten steps across four players — the app, M365 Copilot, the AI agent, and the human reviewer.

1

Upload & identify

A user uploads one or more documents. The system assigns each a unique Document ID.

App
2

Kick off in Copilot

Using M365 Copilot, the user initiates processing for a Doc ID and uploads the file to process. Temporary step until automation is introduced here.

M365 Copilot
3

Extract to JSON

The agent processes the document and extracts its contents into a JSON file, driven by the extraction Skills in the agent.

AI agent
4

Random review draw

A random number is generated. If it matches the “pick” number set in Settings, the document is flagged for human review.

App
5

Business-rule flags

The agent applies a set of business rules from a Skill stored in Dataverse. If any rule is “tripped,” that's another reason to flag for review.

AI agent
6

Human review

The reviewer confirms everything looks good and can make updates if something looks off — then chooses Accept or Reject with Comments.

Reviewer
7

Suggest an improvement

On Reject with Comments, the reviewer is prompted to suggest how the business-rule skill should improve to address what needed fixing.

Reviewer
8

Recommendation generated

Creating a skill update fires a workflow that triggers the skill-update agent to take that input and generate a concrete recommendation for the existing Business Skill.

Workflow → agent
9

Approve the change

The user reviews and approves the recommended change to the skill, updating the record's status accordingly.

Reviewer
10

Skill updated live

The same workflow re-triggers the agent — this time to directly update the Business Skill in Dataverse. Subsequent runs pick up the improved skill.

Workflow → agent

Steps 8 and 10 are the same Power Automate workflow invoking the same Skill-Update Agent. The workflow fires on changes to the Skill Update Request record and selects the agent's prompt based on the record's status: a New request prompts the agent to recommend a fix; an Approved request prompts it to apply the fix to the live Business Skill.
Architecture at a glance

The two halves of the solution

The app owns the human side of the loop. The agent skills own the AI side. They meet at Dataverse — the app sets a status and consumes what the agent writes back.

The Code App src/

Upload, the random review draw, the dynamic field editor, the human review loop, and the Skill Update Request lifecycle UI. Bound to Dataverse.

  • Owns: upload, draw, review, skill-update UI
  • Does not read or extract document content
  • Hands off by setting Processing Status = Queued

The Agent Skills agent/

Natural-language business skills that AI agents discover and follow at runtime, driving the Dataverse MCP server.

  • document-extraction — read file, extract JSON, classify type
  • review-flagging — the policy that decides if review is needed (the skill the loop edits)
  • review-skill-editor — recommends, then applies, edits to the flagging skill
Dataverse

Data model

Four user-owned tables in the HITL Skill Update solution (publisher prefix msfthitl_). The Document record is the spine; the Skill Update Request is the bridge from the app to the agent skills.

The app

Screens

Six role-aware screens. The centerpiece is the Dynamic Field Editor, which infers Fluent UI controls from each document's JSON shape — so the raw JSON is never shown to a reviewer.

All

Dashboard

Counts by processing status, review backlog, recent uploads.

Uploader

Documents

List, filters, and Upload — which runs the random draw and sets Queued.

All

Document Detail

Source-file viewer, status, and the Dynamic Field Editor (read-only unless reviewing).

Reviewer

Review Queue

Flagged, processed, and pending docs. Correct data, Approve, or Reject with Comments.

Reviewer / Admin

Skill Updates

The feedback lifecycle — requests, the agent's recommendation, and status tracking.

Admin

Admin Settings

Edit Review Settings (draw range + trigger) and manage Document Types.
